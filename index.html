Partie 1

L'Anatomie du Web

Comprendre la dynamique entre le Frontend et le Backend.

L'Analogie du Restaurant

Imaginez un restaurant. Le Frontend est la salle à manger : le menu, la décoration, l'endroit où vous interagissez. Le Backend est la cuisine : les ingrédients, le chef, le stockage, cachés à votre vue mais essentiels pour préparer votre commande.

Le Modèle Client-Serveur

Votre navigateur (le client) demande une page au serveur à travers Internet. Le serveur (le backend) traite la demande et renvoie les informations que le navigateur (le frontend) affiche magnifiquement.

Partie 3

Le Backend

Le moteur invisible. La logique métier et les données.

Le Backend s'occupe de la sécurité, du stockage des données et de la logique complexe de l'application.

1. Le Serveur

Un ordinateur puissant toujours connecté, prêt à répondre aux requêtes. Utilise des langages comme Node.js, Python, Java ou PHP.

2. L'Application

Le code qui contient la "logique métier". C'est ici que se font les calculs, l'authentification et le traitement des données.

3. La Base de Données

Le classeur numérique où sont stockées toutes les informations (utilisateurs, produits, etc.). Ex: MySQL, PostgreSQL, MongoDB.

Le HTML est la fondation de tout ce que vous voyez sur le web. Pour bien comprendre, imaginez que le HTML est le squelette d'une maison, le CSS la décoration, et le Backend la plomberie et l'électricité qui font que tout fonctionne en coulisses.

Voici un cours structuré pour vous lancer.

1. Introduction au HTML5

Le HTML (HyperText Markup Language) n'est pas un langage de programmation, mais un langage de balisage. Il utilise des "balises" pour structurer le contenu.

Structure de base d'un document

Chaque page web commence par cette structure :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Ma Première Page</title>
</head>
<body>
    <h1>Bienvenue sur mon site</h1>
    <p>Ceci est un paragraphe de texte.</p>
</body>
</html>

2. Les Balises Essentielles

Voici les éléments que vous utiliserez 90% du temps :

  • Titres : <h1> (le plus important) à <h6>.
  • Listes : <ul> (à puces) ou <ol> (numérotée), avec des <li> à l'intérieur.
  • Liens : <a href="https://google.com">Lien vers Google</a>.
  • Images : <img src="image.jpg" alt="Description">.
  • Conteneurs : <div> (bloc) et <span> (en ligne).

3. Les Formulaires : Le pont vers le Backend

C'est ici que le HTML devient intéressant pour un développeur backend. Un formulaire permet d'envoyer des données utilisateur au serveur.

Exemple de formulaire d'inscription

<form action="/inscription" method="POST">
    <label for="username">Nom d'utilisateur :</label>
    <input type="text" id="username" name="utilisateur" required>

    <label for="email">Email :</label>
    <input type="email" id="email" name="courriel">

    <button type="submit">S'inscrire</button>
</form>

Explications clés pour le Backend :

  • action="/inscription" : C'est l'URL sur le serveur qui va recevoir les données.
  • method="POST" : La manière d'envoyer les données (POST est plus sécurisé que GET pour les formulaires).
  • name="utilisateur" : C'est l'identifiant que le serveur utilisera pour récupérer la valeur saisie.

4. Le Backend : Qu'est-ce qui se passe après le clic ?

Une fois que l'utilisateur clique sur "S'inscrire", le navigateur envoie une requête au serveur.

Exemple concret (Logique Backend)

Imaginons que vous utilisiez Node.js ou Python côté serveur. Voici ce que le backend fait avec votre HTML :

  1. Réception : Le serveur voit arriver une requête sur /inscription.
  2. Traitement : Il extrait les données (ex: utilisateur = "Jean", courriel = "jean@mail.com").
  3. Validation : Il vérifie si l'email est valide et si l'utilisateur n'existe pas déjà.
  4. Stockage : Il enregistre ces infos dans une Base de Données (SQL ou NoSQL).
  5. Réponse : Il renvoie une nouvelle page HTML au navigateur disant "Merci Jean, vous êtes inscrit !".

Comparaison Backend vs Frontend

Caractéristique Frontend (HTML/CSS/JS) Backend (Node, Python, PHP, etc.)
Lieu d'exécution Navigateur de l'utilisateur Serveur distant
Rôle Interface et visuel Logique, Sécurité, Données
Visibilité Code source public Code caché et protégé

5. Résumé du flux de données

  1. HTML affiche le formulaire.
  2. L'utilisateur remplit et valide.
  3. Le navigateur envoie les données via HTTP.
  4. Le Backend traite, interagit avec la base de données.
  5. Le Backend génère une réponse (souvent du HTML ou du JSON).

La Beauté du Web avec CSS

Si le HTML est le squelette, le CSS est la peau, les vêtements et le maquillage de votre site. C'est lui qui donne vie et style à vos pages web !

Voici un cours rapide et structuré pour maîtriser l'essentiel.

1. Introduction au CSS3

Le CSS (Cascading Style Sheets) est le langage de style utilisé pour concevoir et mettre en page des pages web.

Comment intégrer du CSS au HTML ?

  • En ligne (Inline) : <h1 style="color: blue;"> (Déconseillé)
  • Interne : Dans une balise <style> dans votre HTML.
  • Externe (Recommandé) : Via un fichier .css lié comme ceci :
<link rel="stylesheet" href="style.css">

2. Les Sélecteurs CSS

Pour styliser un élément, vous devez lui dire quelle "cible" viser.

  • Balise : p { color: red; } (cible tous les paragraphes)
  • Classe : .ma-classe { font-weight: bold; } (cible les éléments ayant class="ma-classe")
  • ID : #mon-id { margin: 20px; } (cible un élément unique ayant id="mon-id")

3. Le Modèle de Boîte (Box Model)

C'est le concept le plus important en CSS. Chaque élément est vu comme une boîte comprenant :

  1. Content (Contenu) : Le texte ou l'image.
  2. Padding (Marge interne) : L'espace entre le contenu et la bordure.
  3. Border (Bordure) : La ligne entourant la boîte.
  4. Margin (Marge externe) : L'espace invisible poussant les autres éléments.
.boite {
    width: 300px;
    padding: 20px;
    border: 2px solid white;
    margin: 10px auto;
}

4. La Mise en Page : Flexbox

Avant, aligner des éléments était un calvaire. Aujourd'hui, Flexbox rend l'alignement extrêmement simple.

Exemple : Centrer parfaitement un élément

.conteneur {
    display: flex;
    justify-content: center; /* Centre horizontalement */
    align-items: center;  /* Centre verticalement */
    gap: 15px; /* Espace entre les enfants */
}

5. Du CSS au Backend ?

Bien que le CSS soit du Frontend pur, le rôle du développeur complet (Full Stack) inclut souvent l'optimisation côté serveur de ces styles :

  • Minification : Réduire la taille du code CSS (supprimer les espaces) pour un chargement plus rapide.
  • Mise en cache : Configurer le serveur pour dire au navigateur de mémoriser le CSS.
  • Préprocesseurs : Compiler le SCSS en CSS via Node.js ou un outil Backend avant le déploiement de l'application.

Le Cerveau du Web : JavaScript

Si le HTML est le squelette et le CSS la peau, le JavaScript (JS) est le muscle et le cerveau. C'est lui qui fait "bouger" le site, manipuler la donnée et interagir avec vous.

Initialement conçu uniquement pour le Frontend (dans le navigateur), le JS a évolué pour devenir l'un des langages les plus utilisés dans le Backend grâce à Node.js.

1. Les Bases du JS (Frontend)

Côté Frontend, le JS permet d'animer la page, de cacher des éléments, ou encore de vérifier un mot de passe avant de l'envoyer au serveur.

// 1. On déclare des variables pour stocker des données
let prenom = "Alice"; 
const age = 25; // "const" signifie que la valeur (25) ne changera plus

// 2. On crée une action (une fonction)
function direBonjour(nom) {
    alert("Bonjour " + nom + " !"); // Affiche un message à l'écran
}

// 3. On ordonne au navigateur de lancer l'action !
direBonjour(prenom);

2. La Révolution : JS dans le Backend (Node.js)

Créer un serveur Backend demandait autrefois de maîtriser de nouveaux langages (PHP, Java, Python). En 2009, Node.js a tout changé.

Node.js est un environnement qui permet au JavaScript de "sortir" du navigateur pour s'exécuter directement sur la machine du serveur. Le développeur Full Stack peut donc écrire son interface Utilisateur (Front) ET sa logique Serveur (Back) avec le même langage !

3. Comment le JS crée un Serveur Backend ?

Voici un exemple très simple d'un serveur créé en JavaScript. Ce serveur "écoute" en permanence, prêt à répondre aux requêtes Frontend.

// On importe "Express", un outil reconnu pour faciliter la création de serveurs
const express = require('express');

// On initialise l'application Backend
const app = express();

// On crée une "route" : un point d'entrée vers notre Backend 
app.get('/api/citation', function(requete, reponse) {
    // Dès qu'un client Front s'adresse à cette route, on lui renvoie un objet
    reponse.json({ texte: "Le JavaScript gouverne le web !" });
});

// On ordonne au serveur de "tourner" en permanence sur le port 3000
app.listen(3000, function() {
    console.log("🚀 Le serveur JS tourne et vous écoute sur le port 3000");
});

Explications du code étape par étape :

  • require() : Importe l'outil qui facilite le travail (Express). C'est typique d'une approche serveur (modulaire).
  • app.get() : Crée une route (un point d'ancrage avec la méthode GET) à l'URL précise /api/citation.
  • reponse.json() : Répond en convertissant l'information en JSON (la langue universelle des données web).
  • app.listen() : Maintient l'ordinateur serveur "éveillé" à écouter attentivement le port 3000.

4. Le Lien Front / Back (La conclusion)

Voici comment, de son côté, notre page Frontend parvient à récupérer ce que le serveur JS a préparé en utilisant la fonction fetch() ("aller chercher") :

// --- Côté FRONTEND (Dans le navigateur) ---

// 1. On s'adresse à la route définie par notre Backend (l'URL complète)
fetch('http://localhost:3000/api/citation')  
  // 2. Le backend répond, on convertit (décode) la réponse en objet JSON
  .then(reponse => reponse.json())       
  // 3. "donnee" contient enfin notre magnifique objet : { texte: "Le JavaScript..." }
  .then(donnee => {
      // 4. On l'affiche sur l'écran Frontend avec fierté !
      console.log(donnee.texte); // Affiche : Le JavaScript gouverne le web !        
  });

Voilà la magie du web moderne ! Un dialogue fluide, structuré et puissant entre une interface et son moteur en arrière-plan.
